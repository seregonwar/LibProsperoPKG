# LZN Clean-Room Codec

LZN is the native compression workbench for this C++ rewrite. It currently has
two layers:

- `LZN1`: a compact single-frame LZ codec.
- `LZNB`: an indexed block archive reworked from seregonwar's earlier block
  codec design, renamed and cleaned for LibProsperoPkg.

Neither layer copies or translates unlicensed Kraken/newLZ code.

## Frame Format

All integers are little-endian.

| Offset | Size | Field |
|---:|---:|---|
| 0x00 | 4 | Magic `LZN1` |
| 0x04 | 2 | Version, currently `1` |
| 0x06 | 2 | Flags, bit 0 means stored raw payload |
| 0x08 | 8 | Original uncompressed size |
| 0x10 | 8 | Payload size |
| 0x18 | n | Payload |

Compressed payload tokens use one byte per command. Values `0x00..0x7f` encode
literal runs of `token + 1` bytes. Values `0x80..0xff` encode LZ matches with
length `(token & 0x7f) + 4`, followed by a 16-bit backward distance. The encoder
stores raw payloads when compression would not reduce the data size.

The decoder rejects bad magic, unsupported versions/flags, truncated payloads,
extra trailing bytes, invalid match distances, and outputs that do not exactly
match the declared original size.

## CLI

```bash
./build/prosperopkg-lzn compress input.bin output.lzn 2
./build/prosperopkg-lzn decompress output.lzn restored.bin
./build/prosperopkg-lzn bench input.bin 20 2
./build/prosperopkg-lzn block-compress input.bin output.lznb 2 524288
./build/prosperopkg-lzn block-info output.lznb
./build/prosperopkg-lzn block-decompress output.lznb restored.bin
./build/prosperopkg-lzn block-bench input.bin 20 2 524288
```

## LZNB Block Archive

LZNB wraps LZN1 frames in an indexed archive. Each block can be stored raw when
compression would expand it. Every decoded block carries a CRC-32C checksum, and
the public API supports full decompression plus range decompression for future
package-image and random-access workflows.

The old standalone `nm_codec.h` file has been removed from the repository root.
Its useful block-container direction now lives in `lzn_block.hpp` and
`lzn_block.cpp` under GPL-3.0-or-later project headers.

## Local Benchmark

The current benchmark is generated by:

```bash
python3 tools/benchmark_lzn.py --tool build-ninja-release/prosperopkg-lzn --out reports/lzn-benchmark.md
```

See [../reports/lzn-benchmark.md](../reports/lzn-benchmark.md). Current numbers
are an optimization baseline only: zlib still wins ratio on the synthetic
fixtures, and Kraken/newLZ superiority is unproven until a licensed comparison
oracle and real package fixtures are available.
